# OpenClaw 2026.3.24 Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

> **Supersession Note (2026-04-13):** This implementation plan is preserved for historical upgrade execution context. The current source of truth is `docs/superpowers/specs/2026-04-13-multi-kernel-platform-design.md`, with current OpenClaw packaging and activation work aligned to `docs/superpowers/plans/2026-04-13-openclaw-external-node-hard-cut-implementation-plan.md`. Any steps below that assume a bundled OpenClaw runtime layout or bundled Node.js are historical only and must not be resumed as the active implementation path.

**Goal:** Upgrade Claw Studio's bundled OpenClaw runtime from `2026.3.23-2` to stable `2026.3.24` and verify that the embedded desktop integration still works correctly.

**Architecture:** Keep the existing bundled-runtime architecture intact. Update the OpenClaw version pin in the prepare flow, regenerate the shipped desktop runtime artifacts, update every version-sensitive test fixture, and then verify the packaged runtime layout and the Rust embedded-runtime integration path. Do not broaden scope into unrelated bundled-component refreshes or OpenClaw architecture refactors.

**Tech Stack:** Node.js scripts, npm package install/packaged runtime artifacts, Tauri desktop Rust tests, PowerShell, embedded OpenClaw runtime resources
